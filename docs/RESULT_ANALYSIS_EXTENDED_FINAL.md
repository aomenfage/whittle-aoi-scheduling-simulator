# 正式扩展实验结果分析

## 1. 文档说明
本文档基于 `final_extended_suite` 的正式扩展实验结果，对 `WhittleScheduler` 与 4 类基线策略在规模扩展、权重敏感性、链路异构性和峰值风险控制四个维度上的表现进行分析。

本文档使用的数据文件为：

- `results/final_extended_suite/exp_scalability/summary_with_N.csv`
- `results/final_extended_suite/exp_weight_sensitivity/summary_with_weight.csv`
- `results/final_extended_suite/exp_channel_sensitivity/summary_aggregated.csv`
- `results/final_extended_suite/exp_peak_aoi/summary_aggregated.csv`

对应正式扩展实验的默认运行强度为：

- 规模扩展实验：`horizon = 2000`，`num_runs = 5`
- 权重敏感性实验：`horizon = 3000`，`num_runs = 10`
- 链路异构性实验：`horizon = 3000`，`num_runs = 10`
- 峰值 AoI 实验：`horizon = 5000`，`num_runs = 10`

因此，本文档反映的是当前工程下的正式扩展实验结论，而不是 smoke 级快速验证结果。

## 2. 总体结论
从 `final_extended_suite` 的四组结果看，可以先得到三个总体判断：

1. 在权重敏感性、链路异构性和峰值风险控制三类正式扩展实验中，Whittle 都稳定优于 Greedy，且优势方向一致。
2. 在规模扩展实验中，Whittle 在 `N=5` 和 `N=10` 时仍是最优策略，但在 `N=20` 和 `N=50` 时出现异常恶化，显示出当前数值实现存在明显的规模边界。
3. 即便考虑 Whittle 的在线决策开销，其平均单时隙决策耗时仍处于毫秒以下量级，工程上完全可接受；当前问题的主要矛盾不是运行速度，而是大规模下的数值稳健性。

因此，正式扩展实验既支持了 Whittle 方法在异构 AoI 调度中的有效性，也暴露了当前实现从中等规模走向更大规模时的可扩展性瓶颈。

## 3. 规模扩展实验分析

### 3.1 `N=5` 与 `N=10`：Whittle 仍保持最优
平均加权 AoI 结果如下：

#### `N=5`
1. `whittle = 64.9706`
2. `greedy = 68.7151`
3. `round_robin = 86.8630`
4. `periodic = 106.4271`
5. `random = 120.3591`

Whittle 相对 Greedy 的提升约为：

```text
(68.7151 - 64.9706) / 68.7151 ≈ 5.45%
```

#### `N=10`
1. `whittle = 238.5309`
2. `greedy = 252.3244`
3. `round_robin = 333.9761`
4. `periodic = 450.3278`
5. `random = 471.9532`

Whittle 相对 Greedy 的提升约为：

```text
(252.3244 - 238.5309) / 252.3244 ≈ 5.47%
```

这说明在小到中等规模范围内，Whittle 的长期平均调度优势仍然成立，而且优势相当稳定，约维持在 `5%` 左右。

### 3.2 `N=20` 与 `N=50`：Whittle 出现异常恶化
当规模进一步扩大后，结果出现明显拐点。

#### `N=20`
1. `greedy = 976.0438`
2. `round_robin = 1274.3376`
3. `periodic = 1857.5025`
4. `random = 1868.3306`
5. `whittle = 8319.8396`

#### `N=50`
1. `greedy = 5867.8794`
2. `round_robin = 7951.5590`
3. `random = 11078.9117`
4. `periodic = 11655.6855`
5. `whittle = 82891.1799`

Whittle 在这两个点上的结果不是“略差”，而是数量级上明显异常。尤其是结果表中大量业务的平均 AoI 接近 `1001.5`、峰值 AoI 接近 `2001.0`，这基本对应于在 `T=2000` 的时隙长度下，这些业务长期得不到有效更新。

### 3.3 如何解释这一异常
这一现象更适合解释为“当前 Whittle 数值实现和配置在大规模下的稳健性边界”，而不应直接解释为“Whittle 方法在大规模系统中失效”。原因有三点：

1. 在同一批正式扩展实验中，Whittle 在权重、链路和峰值三个维度都持续优于 Greedy，如果理论方法本身失效，通常不会只在规模扩展实验中孤立崩坏。
2. 当前规模扩展实验的 Whittle 依赖有限状态截断与离线索引表预计算；当 `N` 增加到 `20` 和 `50` 时，状态截断 `a_max` 与系统真实 AoI 演化之间可能出现明显失配，导致大 AoI 区间的排序精度下降。
3. 从结果看，异常并不是随机波动，而是部分业务几乎长期不被有效服务，这更像是索引表在高 AoI 区域失去区分度后带来的系统性偏差。

因此，更审慎的表述应当是：

- 当前实现下，Whittle 在 `N<=10` 的正式规模扩展实验中表现良好。
- 当规模进一步放大到 `N=20` 和 `N=50` 时，现有数值截断和索引配置不足以支撑稳定调度，导致结果异常。
- 这暴露的是当前工程实现的规模边界，而不是对 Whittle 理论本身的直接否定。

### 3.4 复杂度角度的补充解释
虽然 Whittle 在大规模下结果异常，但从运行时间上看，它并没有出现不可接受的计算爆炸：

- `N=5`：`avg_decision_time_ms ≈ 0.002929`
- `N=10`：`avg_decision_time_ms ≈ 0.005035`
- `N=20`：`avg_decision_time_ms ≈ 0.009093`
- `N=50`：`avg_decision_time_ms ≈ 0.021025`

即使在 `N=50` 时，平均每时隙决策耗时也仅约 `0.021 ms`。这说明当前问题不在于在线调度太慢，而在于离线索引精度和大状态区间建模不够稳健。

### 3.5 可以支持的结论
规模扩展实验支持如下结论：

1. 当前 Whittle 实现在中小规模系统中具有明确优势。
2. 当前实现尚未完成对更大规模场景的稳健扩展。
3. 后续若要支撑“大规模正式论文结论”，应优先增强 Whittle 的状态截断、索引外推和高 AoI 区域近似处理，而不是单纯继续增加重复次数。

## 4. 权重敏感性实验分析

### 4.1 系统平均加权 AoI
高优先级权重从 `2` 提升到 `10` 时，Whittle 始终取得最低平均加权 AoI：

#### `high_priority_weight = 2`
- `whittle = 24.7852`
- `greedy = 25.4183`
- 相对提升约 `2.49%`

#### `high_priority_weight = 4`
- `whittle = 31.3054`
- `greedy = 33.5559`
- 相对提升约 `6.71%`

#### `high_priority_weight = 6`
- `whittle = 37.0150`
- `greedy = 40.6835`
- 相对提升约 `9.02%`

#### `high_priority_weight = 8`
- `whittle = 42.4752`
- `greedy = 47.2096`
- 相对提升约 `10.03%`

#### `high_priority_weight = 10`
- `whittle = 47.2388`
- `greedy = 53.0777`
- 相对提升约 `11.00%`

### 4.2 结果解读
这一组结果说明，随着高优先级业务权重上升，Whittle 相对 Greedy 的优势不是缩小，而是持续扩大。这说明：

1. Whittle 对权重变化更敏感，能更有效地把长期资源倾斜到真正重要的业务上。
2. Greedy 虽然也会随着 `w_i * A_i` 增大而更频繁地服务高优业务，但它仍然是即时贪心，无法像 Whittle 那样兼顾成功概率、未来收益和状态演化。
3. 当系统目标越来越强调“关键业务优先”时，Whittle 的结构化优势会更明显。

### 4.3 高优先级业务单独指标
若只看最高权重业务的平均 AoI，Greedy 在所有权重点上都略小于 Whittle：

- `w=2`：`greedy = 2.9137`，`whittle = 3.6969`
- `w=4`：`greedy = 2.3719`，`whittle = 2.9669`
- `w=6`：`greedy = 2.1398`，`whittle = 2.7439`
- `w=8`：`greedy = 2.0599`，`whittle = 2.5894`
- `w=10`：`greedy = 2.0003`，`whittle = 2.4509`

这说明在“单个最高优先级业务”这一指标上，Greedy 的即时偏置更激进；但从系统整体加权 AoI 来看，Whittle 始终更优。换句话说：

- Greedy 更像是“把最高优先级业务压得更低”。
- Whittle 更像是“在保证关键业务的同时，兼顾全系统的长期最优”。

因此，这组实验很好地体现了本文方法的目标定位：研究目标是最小化全局加权 AoI，而不是只服务某一个最高权重业务。

## 5. 链路异构性实验分析

### 5.1 平均加权 AoI 结果
在三种链路场景下，Whittle 都稳定排名第一。

#### `uniform`
- `whittle = 48.5496`
- `greedy = 50.8632`
- 相对提升约 `4.55%`

#### `mild_hetero`
- `whittle = 51.7780`
- `greedy = 54.9749`
- 相对提升约 `5.81%`

#### `strong_hetero`
- `whittle = 62.8635`
- `greedy = 69.0178`
- 相对提升约 `8.92%`

### 5.2 结果解读
随着链路从均匀逐步变为强异构，Whittle 的优势不断扩大。这一趋势非常关键，因为它正好对应了 Whittle 的核心建模假设：调度决策不能只看当前 AoI 和权重，还要看不同业务未来成功更新的机会成本。

该结果说明：

1. 当链路成功概率差异较小时，Greedy 和 Whittle 的差距不算大，但 Whittle 已经更优。
2. 当链路异构性增强后，只看 `w_i * A_i` 的贪心策略会忽略“难更新业务”和“易更新业务”的长期差异，性能下降更明显。
3. Whittle 由于在单类问题中显式吸收了有效成功概率信息，因此在强异构链路环境下优势更大。

### 5.3 风险指标补充
系统峰值 AoI 也给出同样结论：

- `uniform`：`whittle = 19.7`，`greedy = 27.2`
- `mild_hetero`：`whittle = 19.0`，`greedy = 30.1`
- `strong_hetero`：`whittle = 23.3`，`greedy = 35.7`

这说明 Whittle 不仅在平均意义下更优，而且在链路不均衡时更能避免某些业务长期“饿死”。

## 6. 峰值 AoI 实验分析

### 6.1 平均加权 AoI
在三类极端风险场景中，Whittle 始终优于 Greedy：

#### `baseline`
- `whittle = 52.2417`
- `greedy = 55.4654`
- 相对提升约 `5.81%`

#### `poor_links`
- `whittle = 68.7011`
- `greedy = 73.9776`
- 相对提升约 `7.13%`

#### `extreme_aging`
- `whittle = 88.9369`
- `greedy = 95.3744`
- 相对提升约 `6.75%`

### 6.2 峰值加权 AoI
如果看峰值加权 AoI，Whittle 同样保持领先：

- `baseline`：`164.65` vs `176.15`
- `poor_links`：`235.65` vs `256.95`
- `extreme_aging`：`308.80` vs `333.25`

### 6.3 系统峰值 AoI
系统峰值 AoI 的结果也一致：

- `baseline`：`whittle = 21.6`，`greedy = 29.9`
- `poor_links`：`whittle = 32.2`，`greedy = 42.1`
- `extreme_aging`：`whittle = 41.2`，`greedy = 54.1`

### 6.4 结果解读
这一组实验最能说明 Whittle 的风险控制价值。Whittle 并不只是把平均值做小，而是在链路变差或老化加剧时，仍能把最坏情况下的信息陈旧度压住。

这对高速铁路监测场景很重要，因为实际系统中最危险的往往不是平均信息稍微变旧，而是少数关键状态长时间没有更新。正式峰值实验结果说明：

1. Whittle 具有更好的尾部风险控制能力。
2. 当环境更恶劣时，这种风险控制优势仍然存在。
3. 因此，Whittle 更适合作为“高可靠状态更新”场景下的核心调度策略。

## 7. 综合讨论

### 7.1 哪些结论是稳定成立的
基于正式扩展实验，目前可以比较有把握地给出以下结论：

1. Whittle 在权重敏感性实验中稳定优于 Greedy，且优势随高优权重上升而扩大。
2. Whittle 在链路异构性实验中稳定优于 Greedy，且优势随异构性增强而扩大。
3. Whittle 在峰值 AoI 实验中同时降低了平均加权 AoI、峰值加权 AoI 和系统峰值 AoI。
4. 因此，在“异构”“不可靠”“重风险”的正式场景下，Whittle 具有很强的方法优势。

### 7.2 哪些结论需要保留边界条件
规模扩展实验则需要更审慎地表达：

1. 当前 Whittle 实现在 `N=5` 和 `N=10` 时表现良好。
2. 当前实现尚不能直接支撑 `N=20` 与 `N=50` 的稳定正式结论。
3. 因此，论文中若引用该组实验，应将其表述为“当前实现的规模边界分析”，而不是“Whittle 理论的大规模负面结论”。

### 7.3 工程角度的启示
从工程角度看，正式扩展实验给出的信息非常清晰：

- 方法层面：Whittle 的建模方向是正确的。
- 实现层面：Whittle 的大规模数值近似仍需增强。
- 部署层面：在线调度速度不是障碍，真正需要优化的是大状态区间下的索引精度。

## 8. 适合写入论文的结论段落
下面这段话可直接作为扩展实验分析部分的初稿：

> 正式扩展实验结果表明，Whittle 指数调度在权重敏感性、链路异构性和峰值风险控制三类场景中均稳定优于 Greedy 等基线策略，并且其优势会随着系统异构性增强而进一步扩大。在高优先级权重从 `2` 增加到 `10` 的过程中，Whittle 相对 Greedy 的平均加权 AoI 改进由约 `2.49%` 提升至 `11.00%`；在链路从均匀到强异构的过程中，该改进由约 `4.55%` 提升至 `8.92%`；在 `poor_links` 与 `extreme_aging` 等高风险场景下，Whittle 在平均加权 AoI、峰值加权 AoI 和系统峰值 AoI 上均保持领先。这些结果说明，Whittle 不仅能够优化长期平均信息新鲜度，还能够有效抑制极端老化风险。另一方面，规模扩展实验显示当前实现的 Whittle 在 `N=20` 和 `N=50` 时出现明显异常，表明现有有限状态截断和索引预计算机制尚存在规模边界。因此，正式扩展实验总体上支持 Whittle 方法的有效性，同时也明确指出了后续工程优化应聚焦于大规模场景下的数值稳健性。 

## 9. 后续建议
若要把正式扩展实验进一步提升为更强的论文证据，建议优先做以下工作：

1. 提高规模扩展实验中的 `whittle_a_max`，并复核 `N=20`、`N=50` 的索引表质量。
2. 针对大 AoI 区域增加索引外推或分段近似，避免高状态区间全部落入同一截断行为。
3. 对异常规模点补充单业务轨迹图，确认是“长期饥饿”还是“局部排序失真”导致性能崩坏。
4. 在修复后重新运行 `exp_scalability`，再决定论文中是否把大规模结果作为正式主结论。
