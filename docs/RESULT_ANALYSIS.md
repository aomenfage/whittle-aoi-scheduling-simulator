# 实验结果分析

## 1. 文档目的
本文档用于对当前工程已经生成的实验结果进行集中分析，说明：

1. 主实验中 Whittle 指数策略与基线算法的相对表现
2. 各类扩展实验所揭示的趋势
3. 当前结果能够支撑的结论边界
4. 现阶段结果的局限性与后续改进方向

本文档基于当前仓库中的已有结果文件撰写，主要参考：

- `results/runner_default/summary_aggregated.csv`
- `results/runner_smoke/summary_aggregated.csv`
- `results/smoke_scalability/summary_with_N.csv`
- `results/smoke_weight/summary_with_weight.csv`
- `results/smoke_channel/summary_aggregated.csv`
- `results/smoke_peak/summary_aggregated.csv`

## 2. 主实验结果分析

### 2.1 主实验结论概览
主实验结果来自 `results/runner_default/summary_aggregated.csv`。在当前默认配置下，Whittle 指数策略在两个标准场景中都取得了最低的平均加权 AoI。

#### 场景一：`N3_demo`

平均加权 AoI 排名如下：

1. `whittle = 17.6633`
2. `greedy = 18.8056`
3. `periodic = 22.9344`
4. `round_robin = 23.2078`
5. `random = 28.7389`

可以看到：

- Whittle 明显优于所有简单基线
- Greedy 是最强基线，但仍略逊于 Whittle
- Random 始终最差，说明简单随机调度无法满足信息新鲜度目标

#### 场景二：`N5_hetero`

平均加权 AoI 排名如下：

1. `whittle = 52.9072`
2. `greedy = 57.4117`
3. `round_robin = 69.4244`
4. `periodic = 81.2806`
5. `random = 98.6411`

这一结果比 `N3_demo` 更有说服力，因为：

- 该场景同时包含异构权重、异构采样率和异构信道
- Whittle 的优势在更复杂的异构场景下被进一步放大

### 2.2 主实验的含义
主实验结果说明：

1. Whittle 策略不仅在简单场景下有效，在中等规模异构场景下也保持稳定优势
2. Greedy 作为短视基线已经较强，但仍无法完全替代基于单类松弛结构构造的 Whittle 指数
3. RoundRobin 和 Periodic 在异构场景中明显吃亏，说明“不考虑状态和成功概率”的静态调度规则难以适应复杂监测系统

### 2.3 主实验可支撑的论文表述
当前主实验足以支撑如下表述：

- 所提 Whittle 指数调度策略在标准高速铁路边缘监测场景下，相较多个经典基线取得了最低的平均加权 AoI
- 该优势在异构监控量场景中更为明显，说明该方法能够有效利用权重差异与链路差异

## 3. 规模扩展实验分析

### 3.1 数据来源与说明
规模扩展结果来自 `results/smoke_scalability/summary_with_N.csv`。

注意：

- 该结果是 smoke 级实验
- `num_runs = 1`
- 用于观察趋势，而不是给出最终论文定量结论

### 3.2 结果趋势
在 `N = 5` 时：

1. `whittle = 60.4833`
2. `greedy = 62.9500`
3. `round_robin = 76.1667`
4. `periodic = 91.1833`
5. `random = 91.4500`

此时 Whittle 仍然最优。

但当规模增大后，结果发生变化：

#### `N = 10`

1. `greedy = 218.9500`
2. `whittle = 266.5417`
3. `round_robin = 311.7417`
4. `random = 345.0583`
5. `periodic = 387.7000`

#### `N = 20`

1. `greedy = 782.7583`
2. `round_robin = 871.1833`
3. `whittle = 1072.6167`
4. `random = 1233.2417`
5. `periodic = 1235.1333`

#### `N = 50`

1. `greedy = 3351.6083`
2. `round_robin = 3532.9167`
3. `random = 3646.3167`
4. `whittle = 3757.2417`
5. `periodic = 4106.4667`

### 3.3 如何解读这一现象
这一结果非常重要，因为它说明：

1. 当前工程中的 Whittle 实现并不是在所有规模下都自动优于 Greedy
2. 在小规模场景中，Whittle 的结构优势能够充分发挥
3. 在更大规模 smoke 场景中，当前数值实现的优势减弱，甚至被 Greedy 超过

### 3.4 可能原因
结合当前工程实现，可以合理推测以下原因：

1. 当前规模扩展实验使用的是 smoke 配置，`num_runs = 1`，统计稳定性较弱
2. `WhittleScheduler` 在扩展实验中采用了较小的 `whittle_a_max`，这是为了控制运行时间，但会降低指数近似精度
3. 当前指数计算采用“数值二分 + 截断 MDP”，而不是解析表达式，因此大规模时近似误差可能更明显
4. Greedy 在大规模异构系统中本身就是一个很强的经验基线，特别是在权重与 AoI 线性相关时

### 3.5 复杂度统计
尽管大规模 smoke 结果中性能未必始终最优，Whittle 在线决策时间仍然保持在很小量级：

- `N=5`: `avg_decision_time_ms ≈ 0.0030`
- `N=10`: `avg_decision_time_ms ≈ 0.0050`
- `N=20`: `avg_decision_time_ms ≈ 0.0107`
- `N=50`: `avg_decision_time_ms ≈ 0.0206`

这说明：

1. 当前实现的在线查表式 Whittle 调度在工程上仍然可运行
2. 即便指数预计算阶段更重，在线阶段复杂度仍近似线性

### 3.6 本节可支撑的论文结论
当前规模扩展实验更适合支撑如下“审慎型”表述：

- 所提 Whittle 方法在小规模和中等规模场景下表现优异，且在线决策时间随规模增长保持可接受水平
- 在更大规模 smoke 级实验中，Whittle 的性能优势未必始终明显，提示后续需要通过更高精度指数计算、更大状态截断或更多重复实验进一步评估其大规模表现

## 4. 权重敏感性实验分析

### 4.1 数据来源
权重敏感性结果来自 `results/smoke_weight/summary_with_weight.csv`。

该实验逐步增加最后一个高优先级业务的权重，主要观察：

1. 系统平均加权 AoI
2. 高优先级业务平均 AoI

### 4.2 主要现象
从系统平均加权 AoI 看，Whittle 在各权重水平下都表现稳定，并且通常优于大多数静态基线。

但如果只看“高优先级业务平均 AoI”，结果更细致：

- 在当前 smoke 结果中，Greedy 往往能把最高权重业务的 AoI 压得更低
- 例如在较高权重设置下，Greedy 对单个关键业务的压制效果比 Whittle 更强

这说明：

1. Greedy 更偏向直接追逐当前最大的 `w_i * A_i`
2. Whittle 更像是在整体平均成本最优意义下做平衡，而不是单独极限压低某一类业务

### 4.3 如何解读
这一现象并不意味着 Whittle 失败，而是反映出两种策略目标侧重点不同：

- Greedy 更偏局部、即时、极端地保护当前最重的业务
- Whittle 更偏全局、长期地平衡所有业务的平均代价

因此在论文中应避免把“关键业务单项 AoI 最小”与“系统加权 AoI 最优”混为一谈。

### 4.4 本节可支撑的论文结论
较稳妥的表述是：

- 当业务权重分布更加不均衡时，Whittle 仍能保持较低的系统平均加权 AoI，表现出良好的全局平衡能力
- Greedy 在某些小规模 smoke 场景下对单个最高权重业务的 AoI 压制更强，说明其对极端权重具有更激进的局部响应

## 5. 链路异构性实验分析

### 5.1 数据来源
链路异构性结果来自 `results/smoke_channel/summary_aggregated.csv`。

### 5.2 平均加权 AoI 结果
在三个链路场景中，Whittle 的平均加权 AoI 分别为：

- `uniform`: `45.8375`
- `mild_hetero`: `53.5375`
- `strong_hetero`: `72.9000`

对应 Greedy 为：

- `46.5625`
- `55.4250`
- `75.8375`

可以看到：

1. 随着链路异构性增强，所有策略性能都下降
2. Whittle 在三个场景中都略优于 Greedy
3. 这种优势在强异构场景下仍然存在

### 5.3 高优先级业务表现
如果只看高优先级业务平均 AoI：

- 在 `uniform` 和 `mild_hetero` 场景下，Greedy 对关键业务略优
- 在 `strong_hetero` 场景下，Whittle 开始体现优势

这说明：

1. 当链路差异不明显时，Greedy 仍可能凭借即时性对关键业务更激进
2. 当链路差异足够明显时，Whittle 由于显式吸收了成功概率信息，其结构优势开始体现

### 5.4 本节可支撑的论文结论
当前结果支持如下表述：

- Whittle 策略能够更好适应异构链路环境，在平均加权 AoI 指标上保持稳定优势
- 该优势在链路异构程度增强时更具有解释性，因为 Whittle 指数中显式融合了更新成功概率

## 6. 峰值 AoI 实验分析

### 6.1 数据来源
峰值实验结果来自 `results/smoke_peak/summary_aggregated.csv`。

该实验重点分析：

1. `system_peak_aoi`
2. `peak_weighted_aoi`

### 6.2 主要结果
#### baseline 场景
- `whittle`: 平均加权 AoI `47.6313`，系统峰值 AoI `13.0`
- `greedy`: 平均加权 AoI `50.1875`，系统峰值 AoI `19.0`

#### poor_links 场景
- `whittle`: 平均加权 AoI `96.0813`，系统峰值 AoI `38.0`
- `greedy`: 平均加权 AoI `106.8313`，系统峰值 AoI `38.0`

#### extreme_aging 场景
- `whittle`: 平均加权 AoI `84.6500`，系统峰值 AoI `26.0`
- `greedy`: 平均加权 AoI `90.5938`，系统峰值 AoI `31.0`

### 6.3 结果解读
这组结果说明：

1. Whittle 不仅在平均指标上更优，在多数极端场景下也能压低系统峰值 AoI
2. 在 `poor_links` 场景下，Whittle 与 Greedy 的系统峰值 AoI 接近，但 Whittle 的峰值加权 AoI 更低
3. 这意味着 Whittle 更擅长压制“高权重业务的极端老化风险”

### 6.4 本节可支撑的论文结论
可写为：

- 所提 Whittle 策略不仅改善了系统长期平均性能，也在峰值 AoI 指标上表现出更强的稳健性
- 该结果说明其适用于对“最坏情况下信息过期风险”高度敏感的高速铁路监测场景

## 7. 决策复杂度分析

### 7.1 在线决策时间
从已有 smoke 结果看：

- RoundRobin、Periodic 的决策时间最小
- Greedy 次之
- Whittle 最大

但当前数值规模下，Whittle 的在线平均决策时间仍然处于毫秒以下甚至更低量级，说明在线查表机制是有效的。

### 7.2 如何在论文中解释复杂度
建议写法：

- Whittle 的预计算成本高于简单基线，但在线阶段仍保持近似线性复杂度
- 对边缘调度器而言，在线决策耗时在当前实验规模下仍具有工程可接受性

## 8. 当前结果的总体结论
综合当前已有实验，可得到以下整体判断：

1. 在标准主实验中，Whittle 是当前最优策略
2. 在链路异构场景和峰值 AoI 场景中，Whittle 优势仍较稳定
3. 在权重敏感性 smoke 结果中，Whittle 更偏向系统整体最优，而 Greedy 更偏向单个关键业务局部压制
4. 在大规模 smoke 场景中，Whittle 不一定持续优于 Greedy，这提示当前实现还有进一步优化空间

因此，更准确的结论不是“Whittle 在所有实验中都绝对最优”，而是：

```text
Whittle 在标准主实验和多数关键扩展实验中表现出稳定优势，
尤其在异构链路和峰值风险控制方面更具结构性优势；
但在更大规模的 smoke 级实验中，其数值实现仍需要进一步优化和更严格验证。
```

## 9. 当前结果的局限性
为了保证结论严谨，当前结果需要明确以下局限：

1. 若干扩展实验仍是 smoke 级验证，重复次数较少
2. 大规模实验中的 `whittle_a_max` 设得较保守，可能影响指数精度
3. 当前 Whittle 指数采用数值近似，而非闭式表达式
4. 权重敏感性实验中的“关键业务 AoI”与“系统加权 AoI”是两个不同目标，不能简单混合解读

## 10. 后续建议
为了把结果进一步提升到论文正式发表水平，建议下一步做以下工作：

1. 对规模扩展实验增大 `num_runs`
2. 提升大规模实验中 Whittle 的状态截断精度
3. 对关键实验结果补充置信区间和显著性分析
4. 将 smoke 结果与 full-run 结果严格分开管理
5. 在图表中明确标注主实验与烟雾实验的区别

## 11. 可直接引用的总结段落
可直接作为论文初稿中的总结段落：

> 实验结果表明，Whittle 指数调度策略在标准对比场景中 consistently 获得最低的平均加权 AoI，并在异构链路和峰值 AoI 控制场景下表现出较强优势，说明该方法能够同时兼顾整体信息新鲜度与极端老化风险抑制。与此同时，规模扩展 smoke 实验显示，当前数值实现的大规模性能仍存在进一步优化空间，特别是在指数近似精度和状态截断设置方面。因此，当前工程结果既验证了 Whittle 方法的核心有效性，也明确指向了后续工程优化与更强实证验证的方向。
